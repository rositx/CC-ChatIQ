Design Section 3: Memoized Streaming Renderer

Detailed optimization schema for token streaming rendering. This prevents CPU usage spikes during active streaming.

React Re-render Optimization Architecture

Current Architecture (Inefficient)

[MessageStore Token Append]
  |
  v
[MessageStream] (Subscribes to token length)
  |
  +--> RE-RENDERS ENTIRE LIST (every token)
       |
       +--> Re-renders all MessageBubbles (unmemoized)

Optimized Architecture (Option A)

[MessageStore Token Append]
  |
  +--> [StreamingAutoScroller] (Leaf component)
  |     -> ONLY this leaf triggers auto-scroll
  |
  +--> [MessageStream] (No token hook)
        -> DOES NOT RE-RENDER
        -> MessageBubbles remain cached
