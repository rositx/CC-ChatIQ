Design Section 1: Backend Presence & Transfer API

Detailed technical specifications for the backend REST endpoints and Redis structures. Please review and approve.

Redis & Database Data Flow
1. Agent WebSocket `/ws/agent/{agent_id}` connects:
   -> Redis: HSET agents:online {agent_id} {timestamp}
2. REST API GET /api/v1/agents/online:
   -> Reads HKEYS agents:online from Redis.
   -> Queries PostgreSQL for active session counts grouped by agent ID.
   -> Returns: [ { "agent_id": "uuid", "active_chats": 3 } ]
3. REST API POST /api/v1/sessions/{session_id}/transfer:
   -> Updates sessions table: agent_id = target_agent_id.
   -> Paves system message in messages table: "You have been transferred to a senior agent."
   -> Broadcasts WebSocket frame to agents & customer socket loops.
Endpoint: GET /api/v1/agents/online

Authorization: Bearer JWT (Agent role)

Response:

[
  {
    "agent_id": "00000000-0000-0000-0000-000000000000",
    "active_chats": 2
  }
]
Endpoint: POST /api/v1/sessions/{session_id}/transfer

Authorization: Bearer JWT (Agent role)

Payload:

{
  "target_agent_id": "d3b07384-d113-4956-a5cc-48a60c8e3100"
}

Response: {"status": "transferred"}
